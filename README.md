# TrafficCam Stabilizer

Пакет для компенсации смещения камер уличного видеонаблюдения (тепловой дрейф, вибрации) и стабилизации заданных пользовательских зон (ROI) с использованием алгоритмов компьютерного зрения (ORB, RANSAC, Гомография).

## Установка

### Режим разработки

```bash
python -m pip install -e .
```

## Использование (CLI)

После установки становится доступна команда traffic_cam.

### track – запуск трекинга и стабилизации

Запускает процесс стабилизации видеопотока (RTSP) или локального видеофайла. Позволяет вручную выделить зону отслеживания и накладывает профилировочный HUD с метриками производительности.

**Параметры:**
- `--source` — путь к видеофайлу или RTSP ссылке (по умолчанию: 0 для веб-камеры)

- `--mask` — путь к PNG маске в градациях серого для исключения нестатичных зон (опционально)

- `--features` — количество извлекаемых ключевых точек ORB (по умолчанию: 500)

- `--width` — ширина кадра для обработки (по умолчанию: исходная)

- `--height` — высота кадра для обработки (по умолчанию: исходная)

- `--points` — количество точек для выделения контрольной зоны (по умолчанию: 4, минимум: 3)
```bash
traffic_cam track --source data/record3.mp4 --width 1280 --height 720 --points 4
```

(В окне программы нажмите SPACE для фиксации эталонного кадра, затем кликните мышкой нужное количество раз, чтобы задать зону отслеживания).

###  analyze – анализ метрик производительности

Читает CSV-файл с собранными метриками работы трекера и строит аналитические графики стабильности системы (FPS, задержки).

**Параметры:**
- `--csv` — путь к CSV-файлу с собранными метриками

```bash
traffic_cam analyze --csv logs/performance_metrics.csv
```

### Справка

Вывод подробной информации о доступных командах и параметрах:

```bash
traffic_cam --help
```

## Логирование и данные

- **Файл логов приложения::** `logs/traffic_cam.log` (уровень DEBUG — подробная информация о работе алгоритмов, скачках времени и ошибках)
- **Консоль:** уровень INFO (краткие системные уведомления о статусах переключения режимов)
- **Метрики производительности:** `logs/performance_metrics.csv` (сырые данные времени выполнения каждого этапа обработки кадра)
- **Графики:** сохраняются в logs/fps_plot.png при запуске команды analyze.
